▶ Quick Start

This tool runs all 14 statistical tests defined in NIST SP 800-22 Rev 1a on a binary sequence. Each test produces a p-value. A p-value below α = 0.01 indicates evidence of non-randomness.

Three steps: ① Enter or generate a binary sequence → ② Adjust parameters if needed → ③ Click Run All Tests

Step 1 — Provide a sequence

Paste any string of 0s and 1s. Spaces and newlines are ignored. Minimum: 100 bits. NIST recommends at least 1 000 bits. Tests 13 & 14 need 10 000+ bits.

Step 2 — Parameters (optional)

Three parameters control specific tests. Default values follow NIST recommendations.

Step 3 — Navigate results

After running, click any test button on the left sidebar to view its chart, statistics, and status on the right panel. The indicator dot shows PASS (green) / FAIL (red) / N/A (grey) at a glance.

🔍 Reading Results

p-value ≥ 0.01 → PASS. No statistical evidence of non-randomness for this test at the 1% level.
p-value < 0.01 → FAIL. The sequence exhibits a detectable pattern.
N/A → Sequence too short for this test.

Tests 10, 13, and 14 report multiple p-values; the PASS/FAIL badge uses the minimum. For any truly random sequence, ~1% of tests fail by chance — an isolated failure is not conclusive.

A p-value very close to 1.0 can also indicate a problem — the sequence may be too "perfectly balanced" (e.g. forced 101010…). True randomness produces p-values uniformly distributed over [0, 1].

🎯 Purpose

Randomness is a foundational requirement in cryptography, secure communications, and scientific research. A sequence that appears random may contain detectable statistical patterns — patterns that can be exploited by an adversary or reveal flaws in a random number generator.

This tool makes the NIST SP 800-22 battery of tests accessible for:

  • Security researchers validating hardware TRNGs and software PRNGs.
  • Students and educators in cryptography or statistics — visual charts make abstract concepts tangible.
  • Developers performing acceptance testing on entropy sources.
  • Anyone curious about whether a given bit sequence looks statistically random.

⚠ Limitations

  • Tests 13 & 14 need ≥ 10 000 bits in practice to reach J ≥ 500 excursion cycles.
  • Test 8 (Maurer) needs ≥ 904 960 bits for the minimum L=6 parameter level.
  • NIST SP 800-22 Section 4 recommends testing ≥ 55 sequences of 1 000 000 bits each for a formal generator evaluation. This tool analyses one sequence at a time.
  • The Generate buttons use Math.random() — not a cryptographically secure RNG. Use for demonstration only.
